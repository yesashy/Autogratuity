How does the GeoJSON import validation system work?

Our GeoJSON import system includes sophisticated validation:

  1. Pre-validation Check - Before processing, the system scans your GeoJSON file to ensure it contains valid location data
  2. Duplicate Detection - The system compares imported addresses with existing records using both exact and fuzzy matching algorithms
  3. Smart Data Extraction - Our parser identifies order IDs, tip amounts, and special instructions from location descriptions
  4. Age-Based Validation - Older delivery records (>14 days) are automatically marked with appropriate status flags
  5. Data Enrichment - Where possible, the system enhances imported data with additional context like normalized addresses

During import, you'll have options to:

  • Skip duplicate entries entirely
  • Update existing records with new information
  • Merge data when both records contain useful information

Our validation system ensures that your imported data maintains its integrity while integrating seamlessly with your existing records.

Troubleshooting

Why aren't my tips being captured automatically?

If automatic tip capture isn't working, check these common issues:

  1. Notification Access - Ensure Autogratuity has notification access permission
  2. Background Restrictions - Check that battery optimization and data saver modes aren't restricting the app
  3. Notification Format - Some delivery apps may have updated their notification format; report this to our support team
  4. Clear Cache - Try clearing the app cache and restarting
  5. App Updates - Ensure both Autogratuity and your delivery apps are updated to the latest versions

Pro subscribers have access to our Robust Capture Service, which uses multiple methods to capture tip data, significantly reducing missed captures.

Why aren't my imported markers showing on the map?

If markers aren't appearing after import, try these solutions:

  1. Verify Import Completion - Check the import results summary to confirm data was successfully imported
  2. Map Refresh - Use the refresh button in the map view to force a reload
  3. Valid Coordinates - Ensure your imported data contains valid GPS coordinates
  4. Filter Settings - Check if you have active filters that might be hiding certain markers
  5. App Restart - Close and reopen the app to refresh all data

If you're still experiencing issues, our enhanced validation system can identify and repair problematic location data. Use the "Validate Data" option in Settings â†’ Data Management.

How can I prevent importing duplicate records?

Our system offers several options for handling duplicates during import:

  1. Pre-Import Scan - Before finalizing any import, the system will scan for potential duplicates and show you a summary
  2. Duplicate Options Dialog - You'll be presented with choices to:
    • Skip all duplicates (fastest option)
    • Update existing records with new information
    • Keep both records (not recommended)
  3. Smart Merge - If you choose to update existing records, our system intelligently combines the most reliable data from both sources

For the most accurate duplicate detection, ensure that addresses in your import data are consistently formatted. Our system uses both exact matching and fuzzy logic to identify potential duplicates based on address similarity.
